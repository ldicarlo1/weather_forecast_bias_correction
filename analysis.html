Observations & Tasks¶
  • Data is hourly collected
  • Time Range: 00z June 1 2021 - 00z July 1 2021
  • GFS & ECMWF are missing the 1st hourly observation
  • Observations need to be preprocessed (i.e. convert all data to the same units, calculate wind speeds, etc)
  • Missing vals in the observation set
  • Only interested in temperature, dewpoint, and wind speed

Max forecast hour dt is 48 (HRRR), which means in order to make a consistent analysis across all 3 models, only the forecast hours of 0-48z can be considered in order to preserve the consistency.

4. Bias Correction of Forecasts¶

In order to bias-correct the forecast data, a machine learning algorithm will be implemented. The reason that a machine learning approach will be taken is in my experience of using a simple mean diurnal cycle bias correction (subtracting the mean forecast by the mean observed between hours 0-23), this method is very difficult to scale up. If data is collected on seasonal/yearly basis, then a seasonal bias correction will need to occur, and even perhaps a yearly bias correction. ML algorithms are flexible, easier to scale up, and more often than not highly effective.

For temperature and wind speed bias correction, I have found that Gradient Boosted Trees or XGBoost perform often better than regular regression techniques. Therefore GBT from Sci-Kit Learn was selected.

Experiment¶

Two variables will be used as inputs, fcst_dt and the forecast variable. The METAR observation variables will be used as the targets. Fcst_dt will represent the temporal variation in the data, particularly how bias increases/decreases as the forecast increases in time. These two variables will be used to train the GBT model. In addition, a bias-correction model will be trained for each variable for each location, and this is to account for the typical differences in bias between two locations (KBOS & KHLC).

My hypothesis is that if GBT is used for bias correction, then temperature/dewpoint/windspeed will all significantly improve in respect to the observational data. However, wind speed will improve the least.

I will first train/test HRRR temperature forecasts for KBOS, then dew point, and windspeed and visualize the results. Then I will perform the training/testing for each model for each location and collect the results.
